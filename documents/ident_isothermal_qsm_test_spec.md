# ТЗ: Валидация и доработка интеграционных тестов идентификации (IdentIsothermalQSM)

## 1. Назначение и цель
Настоящее ТЗ определяет требования к функциональной проверке алгоритмов идентификации параметров изотермической квазистационарной модели трубопровода (по диаметру и коэффициенту гидравлического сопротивления) на основе телеметрических данных СДКУ в репозитории `pde_solvers`.

## 2. Использование градиентных методов (Гаусс-Ньютон)
Процесс идентификации параметров $\theta \in \{\theta_{\text{diam}}, \theta_{\text{fric}}\}$ сводится к минимизации функции невязок давлений $F(\theta) = \frac{1}{2} \| r(\theta) \|^2$:

1. **Вектор невязок $r(\theta)$**:
   $$r_i(\theta) = P_{\text{calc}}(t_i, \theta) - P_{\text{etalon}}(t_i), \quad i = 1 \dots N$$
2. **Численный Якобиан (конечные разности)**:
   $$J_i(\theta) = \frac{\partial r_i}{\partial \theta} \approx \frac{r_i(\theta + \delta) - r_i(\theta - \delta)}{2\delta}$$
3. **Градиент и шаг Гаусса-Ньютона**:
   $$\nabla F(\theta) = J^T r, \quad \Delta \theta = -(J^T J)^{-1} J^T r$$
   После этого выполняется одномерный поиск (Line Search — метод золотого сечения) для подбора длины шага.

## 3. Архитектурные требования и правила (AGENTS.md)
1. **Строгое соблюдение AAA-паттерна**: 
   Все тесты в `ident_quasistatic_isothermal.h` должны содержать явные однострочные комментарии `// Arrange: ...`, `// Act: ...`, `// Assert: ...`. Разделение `Act` и `Assert` в комментариях строго обязательно.
2. **Нейминг тестов**:
   Имена тестов строятся по шаблону `TEST_F(IdentIsothermalQSM, CaseName)`, где `CaseName` пишется в CamelCase без транслита и начинается с глагола в утвердительном формате (`IdentifiesDiameterFromTelemetry`, `IdentifiesFrictionFromTelemetry`).
3. **Запрет на неявную сходимость**:
   Тест не должен проходить без явных операторов `ASSERT_`/`EXPECT_`.
4. **Запрет на `const` у локальных переменных-значений**:
   Локальные переменные внутри тел тестов объявляются без `const` (допустимы только `const&`).

## 4. Требования к проверкам (Assert-секции)
В каждом тесте группы `IdentIsothermalQSM` должны проверяться следующие инварианты:
1. **Код завершения оптимизатора**:
   `ASSERT_EQ(result.result_code, numerical_result_code_t::Converged);`
2. **Физичность полученных параметров**:
   - Для диаметра $\theta_{\text{diam}}$: параметр конечен (`!std::isnan`), $0.5 < d_{\text{rel}} < 1.5$.
   - Для шероховатости $\theta_{\text{fric}}$: параметр конечен, $0.1 < \lambda_{\text{rel}} < 10.0$.
3. **Уменьшение невязки (целевой функции)**:
   Норма невязок $\| r(\theta_{\text{final}}) \|$ должна быть строго меньше начальной нормы $\| r(\theta_{\text{initial}}) \|$.
4. **Вывод отчётов (для Printer-тестов)**:
   Проверять факт физического создания CSV-файлов отчётов (`ident_diff_press.csv`) через `std::filesystem::exists`.
