# ТЗ: Синтетический эталонный тест расчёта давления IdentIsothermalQSM

## 1. Назначение и цель

Настоящее ТЗ определяет требования к **синтетическому регрессионному тесту**, который фиксирует эталонное поведение изотермической квазистационарной PQ-модели трубопровода по расчётному давлению на выходе ЛУ.

**Цель:**
1. Использовать реальные параметры трубопровода (профиль высот $Z(x)$, диаметр $D$) и реальные технологические графики прокачки ($Q_{in}$, $P_{in}$, $\rho$, $\nu$).
2. Сравнивать текущий расчёт PQ-модели напрямую с зафиксированным синтетическим эталоном давления $P_{out}^{\text{synthetic}}(t)$ из файла `synthetic_pout.csv`.
3. Задать **коридор безопасности** в процентном выражении, который позволяет улучшать физику модели, но защищает от регрессий.

---

## 2. Архитектурное решение

### 2.1 Прямой вызов PQ-модели (без обёртки идентификации)
Тест вызывает PQ-модель **напрямую** через `quasistatic_batch`, минуя обёртку `ident_isothermal_qsm_pipe_parameters_t`. Это обеспечивает:
- Независимость от API идентификации (Якобиан, Line Search, оптимизатор).
- Минимальное время выполнения (нет лишних слоёв абстракции).
- Тест проверяет **физику расчёта давления**, а не работу оптимизатора.

### 2.2 Статический синтетический эталон
Файл `synthetic_pout.csv` сгенерирован **однократно** эталонной моделью и зафиксирован в репозитории. Повторная генерация в C++ коде **отсутствует**.

### 2.3 Коридор безопасности (Safety Corridor)
Для каждой минуты ($N = 43\,200$ временных шагов) рассчитывается относительная невязка:
$$\delta_i = \frac{|P_{\text{текущий C++ код}}(t_i) - P_{\text{synthetic}}(t_i)|}{P_{\text{synthetic}}(t_i)} \times 100\%$$

- **MAPE (средняя процентная ошибка)**: $\le \mathbf{0.1\%}$ (допускаются точечные улучшения формул).
- **Max PE (пиковая относительная ошибка)**: $\le \mathbf{0.5\%}$ (защита от грубых ошибок и регрессий).

---

## 3. Логика и этапность выполнения теста (AAA)

Тест `CalculatesSyntheticPressureEtalon` строится строго по паттерну **Arrange-Act-Assert**:

### 3.1 Подготовка данных (Arrange)
1. **Геометрия и свойства трубы:** Загружаются реальные параметры участка трубопровода через `prepare_pipe()`.
2. **Временные ряды прокачки:** Извлекаются временные метки $t_i$ и входные краевые условия ($Q_{in}, P_{in}, \rho, \nu$) за 1 месяц ($43\,200$ минут) через `prepare_real_data()`.
3. **Синтетический эталон:** Из статического файла `synthetic_pout.csv` через `load_synthetic_etalon()` считывается эталонный массив давлений.
4. **Валидация длины выборки:** Проверяется совпадение количества элементов эталонного массива и количества временных шагов (`ASSERT_EQ`).

### 3.2 Расчёт модели (Act)
1. **Прямой запуск PQ-соловера:** Создаётся задача `isothermal_quasistatic_PQ_task_t` и через `quasistatic_batch` с коллектором `isothermal_qsm_batch_Pout_collector_t` рассчитываются выходные давления текущего C++ кода $P_{\text{calc}}(t_i)$.
2. **Вычисление процентных невязок:** 
   - Вычисляется вектор абсолютных невязок $|P_{\text{calc}}(t_i) - P_{\text{synthetic}}(t_i)|$.
   - Формируется вектор поэлементных процентных ошибок $\delta_i$.
   - Рассчитываются итоговые метрики: среднее значение `MAPE` ($\bar{\delta}$) и максимум `Max PE` ($\max \delta_i$).

### 3.3 Проверка условий и диагностика (Assert)
1. **Проверка конечности:** Убеждаемся, что значение `MAPE` не содержит `NaN`.
2. **Контроль коридора безопасности:** 
   - Сравниваем `MAPE` с верхним порогом $0.1\%$ (`ASSERT_LE`).
   - Сравниваем `Max PE` с верхним порогом $0.5\%$ (`ASSERT_LE`).
3. **Диагностические сообщения:** При превышении порогов выводится явный вывод измеренных значений и величины превышения для удобства отладки.
